## 1. 当前分类工程的分层结构

当前分类工程不再是一条单线，而是四层并存：

1. `PIA-Operator` 现役主线
2. `dynamic manifold classification` 动态分类线
3. 并行研究分支
4. 历史背景层

当前最重要的整理原则是：

- `PIA-Operator` 负责当前主叙事
- `dynamic manifold` 负责平行系统路径
- `R0 / P0b-lite / P1a` 属于当前主线后的分流诊断与前沿探针
- `SCP branch / dual-stream / 旧 Route B` 都不应再误读成默认主线

`P0a.1` 尾部已经真实跑通的核心组合是：

### 2.4 当前主问题

当前主问题已经不是：

- 算子能不能跑
- 判别轴能不能学出来

而更像是：

**学出来的判别轴，还没有稳定翻译成一个对终端始终有利的局部推进场。**

当前最直接的结构证据仍是：

- [geometry_visual_summary.md](/home/THL/project/MTS-PIA/out/_active/verify_route_b_pia_operator_p0a1_c3_20260401_smoke/VIS/natops_seed1/geometry_visual_summary.md)

它说明：

- `C3` 已显著把模板方向拉向 pair normal
- 但 `response_vs_margin_correlation` 仍没有自动变成理想单调关系

所以当前真正的瓶颈更接近：

- `axis -> force`
- `global -> local execution`
- `operator scope`

而不是“判别方向完全学不出来”。
