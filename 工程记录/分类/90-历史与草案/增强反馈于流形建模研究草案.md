# 增强反馈于流形建模研究草案

更新时间：2026-03-29

## 1. 当前框架升级方向的定位

这条研究线不是对现有工作的降级补丁，而是一个明确的框架升级方向。  
当前要做的不是弱化它，而是把它收成一个更可证伪、可归因、可逐层落地的升级路线。

当前真正要回答的问题不是：

- 在固定流形里还能怎么调 `axis-2`
- 在固定 `z-space` 里还能怎么挪点

而是：

**增强结果是否能经过筛选后回流，进入流形构造闭环，直接改善参考中心、模板空间与风险几何，从而让同一个增强链在下一轮工作得更健康、更有效。**

一句话说：

> 当前研究对象不是 `augmentation on manifold`，而是 `filtered augmentation feeds manifold construction`。

这条线应被明确写成：

- **面向下一代主线的框架升级方向**
- 当前先以分阶段 probe 方式推进
- 分阶段推进是工程路径，不是概念降级

当前这样做的目的，是把升级过程拆成可验证层级，先确认：

- 增强是否只是在固定流形里扩点
- 还是已经提供了足够可靠的几何信息，值得反过来参与流形构造

需要额外说明的是：

- 这条升级方向最终希望触及流形的 `basis / skeleton`
- 当前第一阶段最现实的落点是 `re-basis + re-centering`
- 严格说这还不是完整的 `metric learning`
- 但它已经不再是“在死空间里换原点”

## 2. 为什么原版草案需要修正

### 2.1 原版 M0 容易产生假阴性

如果把全部 `z_aug` 无差别回流去更新中心，那么结果很可能只是：

- 中心发生偏移
- 增强云团变厚
- 但没有真正带来更好的几何结构

原因很直接：

- 当前增强主要仍沿既有模板轴做仿射偏移
- 全量回流不会自动产生新的结构信息
- 反而容易把“增强器自己的输出偏好”重新写回参考几何

因此必须写成硬约束：

**禁止全量增强样本直接进入 center feedback。**

### 2.2 原版把“中心更新”和“骨架更新”拆开了

当前 `mean_log_train` 同时扮演两种角色：

- log-Euclidean 参考中心
- centered representation 的坐标原点

如果直接说“更新中心后流形变好了”，就会混淆：

- 是流形参考几何真的更适合增强链
- 还是只是坐标参考点变了

真正的问题是：

- 只更新中心，最多改变参考原点
- 不重新 `fit TELM2`，就不会真实改写模板 `W`
- 不改 `W`，就不会真实改写轴身份和 basis 结构

因此如果要让增强真正反馈到流形构造，中心与模板不该在第一步被机械拆开。

更准确的第一阶段对象应该是：

**Densified Manifold Re-basis**

也就是：

- feedback pool 先帮助我们定义“新领土”
- 再在这个致密化池上同时更新参考中心与模板骨架

### 2.3 原版缺少 Feedback Pool

这条是当前最关键的结构修正。

`LIRF` 给出的真正启发不是“做循环”本身，而是：

- 生成后要有 correction
- correction 后要有 admission
- 只有通过阈值的样本才参与 densification / feedback

因此当前必须新增一层：

**Feedback Pool Construction**

它不是下游训练增强池，而是：

**专门服务于流形构造反馈的安全池。**

这一层要明确回答：

- 哪些 `z_aug` 能回流
- 录取规则是什么
- 是按 `distortion`、机制量、类条件安全性，还是最小组合规则录取

当前必须明确区分：

- `Train Pool`
- `Feedback Pool`

这两个池不能默认相同。

更准确地说，这一层不是普通 admission，而是：

**task-constrained feedback admission**

### 2.4 原版验证目标不对

M0 第一版如果只换参考中心，而不引入新的增强链路，那么用它直接去比较：

- `raw baseline`

在当前主链下并不合理。

因为：

- 只换中心，不会凭空给 raw 终端创造新信息
- raw baseline 本身并不直接感知“参考中心是否更合理”

M0+ 的正确验证目标应该是：

- `旧 basis/旧中心 + 固定 operator`
- `新 basis/新中心 + 同一个 downstream chain`

比较的是：

**同一增强链在新旧参考几何下的结构变化、机制变化和最终稳定性变化。**

同时第一轮必须避免一个混淆：

- 不能一边做 `densified pool` 回流
- 一边又把同一批 densified 样本直接拿去扩充最终训练集

否则你分不清：

- 是 `re-basis` 生效
- 还是“只是训练样本变多了”

## 3. 修正后的总结构

当前蓝图应收紧成下面五层：

### B0：Manifold Diagnostics

先建立“流形有没有变好”的观测层。

这一层的职责不是做最终裁判，而是：

- 看空间变化
- 看失败模式
- 看增强是否只是让空间变厚

至少包括三类诊断：

#### A. 低维投影诊断

至少比较：

- 原始表示
- 当前向量版 operator
- log-Euclidean 自洽版 operator
- 后续 re-basis 版

观察：

- 类内连续性
- 类间分离
- 是否只是整体变厚
- 是否出现错误桥接

#### B. SPD 椭球诊断

借 `MENDR` 的思路，把 SPD / log-cov 表示当成几何椭球来解释。

观察：

- 主轴长度变化
- 主轴方向变化
- 类条件椭球群的形变模式

#### C. 邻域/连通性诊断

借 `LIRF` 的 densification 思路，但用于分类几何诊断。

观察：

- 类内邻域是否更密
- 类间是否被错误桥接
- 是结构变好，还是只是空间更厚

### B1：Feedback Pool Construction

这一层是当前蓝图里必须新增的硬模块。

目标不是“多保留增强样本”，而是：

**只让足够安全、足够有结构意义的增强样本参与流形构造反馈。**

第一版建议最小规则：

- 先保留低 distortion 样本
- 再叠加一个最小安全条件：
  - `flip` 不恶化
  - 或 `fidelity/cond_A` 不过差
- 必要时做简单类平衡

当前必须明确：

- `Train Pool`：给下游训练用
- `Feedback Pool`：给中心/模板/风险几何更新用

这两个池不能默认相同。

这一层的输出不是最终训练增强集，而是：

- 一个可审计、可复现的反馈池构造规则
- 以及被 admitted 的 `feedback_pool`

### B2：M0+ Densified Manifold Re-basis Probe

当前真正有推进意义的第一阶段，不再是 center-only M0，而是：

**M0+ = Densified Manifold Re-basis Probe**

其核心逻辑是：

- 先用当前最优 `PIA Core` 生成候选增强样本
- 用 `Feedback Pool` 只保留安全、可信的边界延伸
- 将 `orig_z + feedback_pool` 合成 `densified pool`
- 在这个致密化池上同时更新：
  - 参考中心
  - `TELM2` 模板 `W`

也就是说，第一阶段就直接触及流形的骨架，而不把“更新中心”和“更新模板”机械拆开。

建议把执行步骤写死成下面五步：

1. `Generation & Filtering`
   - 用当前最优 operator 生成 `z_aug`
   - 通过 `distortion / flip / fidelity` 做安全筛选
   - 得到 `feedback_pool`
2. `Densification`
   - 构造 `densified_pool = orig_z + feedback_pool`
3. `Re-centering`
   - 在 `densified_pool` 上重算参考中心
4. `Re-basis`
   - 在 `densified_pool` 上重新调用 `TELM2.fit()`
   - 得到 `W_new`
5. `Re-augmentation under new basis`
   - 用 `W_new` 和新中心，在原始 `orig_z` 上重新生成一轮增强

其中第 4 步在第一版实现时必须优先采用：

- **受限重演化（constrained re-basis）**

而不是：

- **完全自由重学（fully free re-fit）**

第一版至少应满足下面三条中的一条，最好同时满足两条：

- 以 `W_old` 为 warm-start
- 限制 `TELM2` 的更新轮数
- 显式记录并约束 `W_old -> W_new` 的子空间漂移幅度

这样做的原因很明确：

- 避免增强样本沿旧 `W` 拉出后，又被完全自由重学再次强化，形成模板回音壁
- 避免把 `TELM2` 重新追逐被人为放大的方差方向，误判成 basis 重构成功
- 保证第一轮 `M0+` 仍然可归因、可审计

这里最后一步很关键：

- 第一轮验证应尽量固定 downstream chain
- 用 `W_new` 重新增强原始 `orig_z`
- 而不是直接把 `densified_pool` 本身送去下游训练

这样才能更干净地验证：

- 是 `basis` 被重构后更有效
- 还是只是因为样本数变多

### B3：M0+ Validation

M0+ 的验证原则必须写死：

**验证对象不是 raw baseline，而是固定 downstream chain 下，旧参考几何与新参考几何谁更有效。**

也就是固定：

- downstream bridge / evaluator
- operator 族
- 其余比较口径

比较：

- `旧 basis/旧中心`
- `新 basis/新中心`

至少看四类结果：

#### 骨架层

- `W_old` vs `W_new` 的子空间变化
- 轴排序是否重排
- 各轴投影能量是否改写
- `W_old -> W_new` 的子空间漂移是否处在可解释范围内

#### 结构层

- centered z-space 的类内/类间统计
- 邻域连通性
- SPD 椭球群是否更合理

#### 机制层

- `flip`
- `margin_drop`
- `classwise_covariance_distortion`
- `cond_A`

#### 最终层

- 在同一 downstream chain 下，最终 raw 指标是否更稳
- 是否更接近共同候选点

### B4：Only if positive -> Risk Geometry Feedback / Multi-round Re-basis

只有当 M0+ 稳定成立，才允许进入：

- 风险几何反馈
- 或真正的 multi-round re-basis

在此之前，不进入更深层闭环。

否则极易进入：

- 模板回音壁效应
- 自我实现的结构偏移
- 不可归因的大闭环

## 4. 这条研究线与论文的对应关系

当前最相关的两篇仍是：

- [MENDR.pdf](/home/THL/project/MTS-PIA/archive/reference_materials/相关论文/MENDR.pdf)
- [LIRF.pdf](/home/THL/project/MTS-PIA/archive/reference_materials/相关论文/LIRF.pdf)

### 4.1 MENDR 的作用

`MENDR` 对当前最有价值的不是具体模型，而是：

- 把 SPD / manifold representation 本身当成可解释对象
- 允许直接分析 embedding 几何
- 允许用椭球与低维投影理解结构变化

因此当前蓝图里的 `B0 diagnostics` 与 `SPD 椭球解释`，可以明确视为对 `MENDR` 思路的吸收。

### 4.2 LIRF 的作用

`LIRF` 对当前最重要的不是“循环”本身，而是：

- `generation -> correction -> admission -> densification`

因此当前蓝图里新增的 `Feedback Pool Construction`，本质上是在吸收 `LIRF` 的 `correction + threshold admission + densification` 逻辑，只是把它转译成当前分类几何管线里的版本。

## 5. 当前阶段的硬约束

下面几条应视为这条研究线的正式规则：

### 规则 1

**禁止全量增强样本直接进入 center feedback。**

### 规则 2

**第一阶段不再停留在 center-only M0，而直接定义为 `M0+ Densified Manifold Re-basis Probe`。**

### 规则 3

**M0+ 的验证目标不是 raw baseline，而是“新 basis/新中心下同一 downstream chain 是否更有效、更健康”。**

### 规则 4

**Feedback Pool 必须独立于 Train Pool 定义。**

### 规则 5

**第一轮必须隔离 `re-basis` 效应，不允许把 densified pool 直接当作额外训练样本混入最终训练集。**

### 规则 6

**M0+ 第一版中的 `Re-basis` 必须优先采用受限重演化，而不是完全自由重学。**

### 规则 7

**M0+ 的成功判据必须同时覆盖反馈池稳定性、骨架层变化、结构层变化、机制层改善。**

### 规则 8

**M0+ 未成立前，不进入风险几何反馈或多轮 re-basis。**

## 6. 当前最合理的推进顺序

当前这条研究线应按下面顺序推进：

### B0：Manifold Diagnostics

先把空间变化看见。

### B1：Feedback Pool Construction

定义谁有资格回流。

### B2：M0+ Densified Manifold Re-basis Probe

用反馈池重建中心与 basis。

### B3：M0+ Validation

固定 downstream chain，比新旧参考几何。

### B4：Only if positive -> Risk Geometry Feedback / Multi-round Re-basis

只有 M0+ 成立，才进入更深层闭环。

## 7. 当前工程上的最小实现原则

如果后面真的开做，当前最稳的工程原则是：

1. 不打断当前分类主线 runner
2. 单独新增最小 probe runner
3. 先只做 `NATOPS + SCP1`
4. 先只做 `Feedback Pool + densified re-basis`
5. 不和当前第二包 `axis refine` 混成一个系统

换句话说：

- 这是框架升级方向的分阶段推进
- 当前以研究 probe 形式落地，是为了控制归因风险
- 不是对方向本身的降级
- 也不是立刻开启一个无法审计的新统一壳

## 8. 当前阶段的判断标准

这条研究线当前不要求立刻提高 raw 终点分数。

M0+ 第一版的成功判据必须至少分成四类：

### A. 反馈池本身稳定

- `Feedback Pool` 能稳定构造
- 录取样本不是极少数偶然点
- 各 seed 下的反馈池规模与构成不至于完全漂移

### B. 骨架层发生真实变化

- `W_old` 与 `W_new` 的子空间变化不是接近零
- 轴排序或投影能量分配出现可解释改写
- basis 变化不是纯噪声抖动

### C. 结构层出现一致变化

- 类内/类间统计出现一致变化
- 邻域连通性更合理
- centered log-matrix 分布不只是整体变厚
- 若接入可视化，SPD 椭球群几何出现可解释变化

### D. 在固定 downstream chain 下，增强链表现更健康

- `flip` 更低或不恶化
- `margin_drop` 更健康
- `classwise_covariance_distortion` 更低或更稳
- `cond_A` 更低或更稳
- 必要时再看最终 raw 指标是否更稳或更接近共同候选点

只有当 A/B/C/D 至少同时给出一致正信号，才值得继续进入风险几何反馈层。

如果连这一层都没有稳定信号，就说明：

- 当前增强结果尚未提供足够可靠的 basis 重构信息

---

## 9. 当前一句话结论

这条研究线的方向没有问题，但必须从：

- “增强样本回流更新中心”

修正为：

- **经过非线性筛选的反馈池，参与参考中心重估，并在固定增强链下验证其是否真的改善流形参考几何。**

因此，当前真正的第一步不再是宽泛的 `Center Feedback Probe`，而是：

**`Feedback Pool Construction + M0+ Densified Manifold Re-basis Probe`。**
