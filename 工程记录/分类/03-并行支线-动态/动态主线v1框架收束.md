# 动态主线 v1 框架收束

更新时间：2026-03-30

## 一、当前收束结论

当前不要再把所有动态研究支线继续挂在默认链上。

当前应正式切成两层：

### A. 动态主线 v1

只保留最稳定、最像系统升级的部分：

- 输入：`raw`
- 表示：`dense z_seq`
- 终端：`dynamic_minirocket`
- 参考：
  - `static_linear`
  - `raw_minirocket`

一句话记法：

`raw -> dense z_seq -> dynamic_minirocket`

### B. 研究外挂层

以下内容全部降级为：

- 可插拔研究模块
- 结构证据层

而不是默认主链的一部分：

- `T2a default`
- `T3 shared rebasis`
- `T4b window-level object`
- `T4b radial / margin`
- `T5`
- `T6a-1`
- `T6b`
- `T7a`

这不是否定这些工作，而是承认：

> 它们当前更像“帮助定位瓶颈的研究证据”，还不是“能稳定抬主线的系统部件”。

---

## 二、为什么要这样收

### 1. 当前问题不是“没试够”，而是主链被研究协议淹没

前期动态线系统地试过了：

- `object`
- `gate`
- `policy`
- `container`
- `terminal`

但这些层大多通过：

- 冻结
- 切换
- 局部组合

来推进。

这使得工程越来越像：

- 一组研究性 probe

而不是：

- 一个端到端工作的默认框架

### 2. `dense z_seq` 更像主干升级，不像边角修补

`T8-pre` 当前改变的是：

- 时间分辨率
- 轨迹表示密度
- 下游终端的可用性

这属于表示层主干升级，不是某个局部规则。

### 3. `dynamic_minirocket` 更像主线终端，不像研究外挂

它回答的是：

> 当前动态表示本身，交给更稳的 trial-level 序列终端后，能走到哪一步。

这属于主系统问题。

---

## 三、动态主线 v1 的默认协议

以下内容应从“可调参数”降级为“框架默认协议”：

- `spd_eps = 1e-4`
- `prop_win_ratio = 0.20`
- `force_hop_len = 1`
- `padding_mode = edge`
- `target_len_mode = train_max_len`
- `dynamic_minirocket` 只吃 `z_seq`
- `static` 不做伪序列 `MiniROCKET`
- `val/test` 只走原始 trajectory
- 当前主验证站点先固定为：
  - `NATOPS`
  - `SelfRegulationSCP1`

这些协议一旦写死，动态主线 v1 才像主线，而不是继续滑回参数收口。

---

## 四、研究外挂层的三级分组

### 1. 默认外挂候选

这些仍保留为后续最可能回到主链的第一层外挂：

- `T2a default`
- `T3 shared rebasis`
- `T4b window-level object`

说明：

- `T2a` 更像默认增强生成器外挂
- `T3` 更像 `NATOPS` 上已成立的几何外挂主件
- `T4b` 更像 object-level 结构证据外挂

### 2. 结构证据层

以下不再进入默认链，只保留为：

- 边界实验
- 容器否证
- 角色分离证据

具体包括：

- `T5`
- `T6a-1`
- `T6b`
- `T7a`

### 3. 下一代主线候选

`T8` 系列保留为：

- 未来可能从 v1 跃迁到 v2 的方向

包括：

- 更强终端统一
- 非线性/双流容器
- 未来可能的轨迹动力学干预

---

## 五、当前判断

当前最合理的工程推进方式不是：

- 继续把所有研究支线叠进默认链

而是：

1. 先把动态主线 v1 收口为：
   - `raw -> dense z_seq -> dynamic_minirocket`
2. 用 `static_linear / raw_minirocket` 作为固定参考
3. 把 `T2a/T3/T4b` 收成第一层可插拔外挂
4. 把 `T5/T6/T7` 留在结构证据层

---

- `NATOPS + SCP1`

再根据 formal 结果决定：

- 是否把 `T3/T4b` 中的某个外挂重新接回 v1
- 还是直接进入 `T8` 的下一阶段

- `NATOPS` 上，`dense_dynamic_gru` 仍明显更强
- 因而当前主线更像：
  - `dense dynamic mainline`
  - 加上 `terminal split evidence`

也就是说：

> 当前动态主线 v1 已经可以收口为一个更小的系统形态，但它还不是完全统一的单终端闭式答案。
