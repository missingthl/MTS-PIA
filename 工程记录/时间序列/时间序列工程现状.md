# 时间序列工程现状

更新时间：2026-03-13

## 1. 定位

当前时间序列方向是工程中的一条独立副线，目标不是替代现有分类主线，而是验证工程是否能够稳定扩展到标准 forecasting 任务。

当前定位：

- `classification mainline`：`PIA + 几何特征 + Gate + LinearSVC`
- `forecast sideline`：`ETT raw forecasting probe`

这两条线当前必须严格分离：

- 不共享 split 协议
- 不共享评测指标
- 不混合结论

## 2. 当前 forecasting 副线的数据流

当前已落地的 forecasting probe 面向 ETT 数据集，先从 `ETTh1` 开始。

稳定数据流如下：

1. 从 `data/MTSFC/EET/ETTh1.csv` 读取连续时间序列
2. 使用官方小时级 split：
   - train：前 12 个月
   - val：后续 4 个月
   - test：最后 4 个月
3. 只在 train 上拟合：
   - 输入特征 scaler
   - 目标 `OT` scaler
4. 构造 forecasting 样本：
   - `X_t`：过去 `L` 步的多变量输入
   - `y_t`：未来 `H` 步的 `OT`
5. 模型输出未来 `H` 步预测
6. 预测结果做 inverse-transform，回到原始 `OT` 尺度
7. 在原始尺度上计算 `MAE / RMSE`

当前锁定的是：

- 多变量输入，单目标输出
- `OT` 历史值包含在输入窗口中
- `direct multi-output forecasting`

## 3. 当前模块

### 3.1 ETT Loader / Sample Builder

- 路径：`datasets/forecast_ett.py`
- 作用：
  - 读取 ETT CSV
  - 应用官方 split
  - 构造 `(X, y)` forecasting 样本
  - 管理 train-only scaler 与 inverse-transform

关键约束：

- 不允许 shuffle
- val/test 只允许使用过去历史上下文，不允许未来泄漏
- metrics 必须在逆归一化后计算

### 3.2 Forecast Probe Runner

- 路径：`scripts/run_forecast_probe_ett.py`
- 当前职责：
  - 驱动 ETTh1 forecasting probe
  - 支持：
    - `persistence`
    - `ridge`
    - `elm`
    - `dlinear`
    - `nlinear`
  - 落盘：
    - `metrics.json`
    - `run_meta.json`
    - `summary_per_run.csv`
    - `summary_agg.csv`
    - `summary_compare_vs_persistence.csv`

### 3.3 当前 baseline 定义

当前已验证的 baseline / plugin 顺序：

1. `Persistence`
   - 直接把最后一个观测到的 `OT` 重复到未来
2. `Ridge Regression`
   - 把过去窗口展平
   - 直接回归未来 `H` 步 `OT`
3. `ELM Regression`
   - 同样基于展平输入
4. `DLinear`
   - 当前最强 validation baseline family
5. `NLinear`
   - 当前最强 test baseline family
6. `PIA-lite raw history plugin`
   - 只作为训练期 history augmentation 插件
   - 当前只做了 `F2-A` 最小验证

## 7. PIA Dynamics Probe 状态

为验证“PIA 是否可以从模板中心/主方向提取器，过渡到局部动态基/游走量框架”，当前已新开一条独立 probe 线：

- `datasets/forecast_zseq.py`
- `scripts/run_pia_dynamics_probe.py`
- 输出目录：`out/pia_dynamics_probe/`

这条线与现有 raw forecasting sideline 分离，当前只做：

- 数据集：`ETTh1`
- 表示空间：`z-space`
- 最小状态构造：
  - 先对 ETTh1 的 7 维原始变量做 train-only 标准化
  - 再用长度 `L=96` 的滑窗构造协方差
  - 通过 `log-center(train only)` + 上三角向量化得到 `z_t`
- 任务：
  - 一步预测：`z_t -> z_{t+1}`

当前已完成 `V0`：

- `z-persistence`：`z_hat_{t+1} = z_t`
- `z-ridge`：`z_hat_{t+1} = A z_t + b`

结果：

| stage | model | z_MAE(test) | z_MSE(test) | z_MAE(val) | z_MSE(val) |
| --- | --- | ---: | ---: | ---: | ---: |
| V0 | z-persistence | 0.007548 | 0.0002364 | 0.008993 | 0.0003963 |
| V0 | z-ridge | 0.008072 | 0.0002423 | 0.009948 | 0.0004263 |

当前判断：

- 在当前最小 `ETTh1 -> z_t -> z_{t+1}` 构造下，`z-ridge` 没有优于 `z-persistence`
- 因此按当前 probe 规划中的 stop rule：
  - 不继续进入 `V1 state-vs-delta directions`
  - 不继续进入 `V2 basis transition`

这说明：

- 当前这套最小 z-state 构造在 ETTh1 上，表现为非常强的“一步近似恒等推进”
- 但还不能据此直接否定“动态基”方向本身
- 更准确的说法是：
  - **当前 ETTh1 的最小 z-space 构造，没有给出足够强的 V0 信号**
