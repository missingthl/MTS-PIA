## 1. 当前阶段定位

当前分类工程已经进入：

**借助外部宿主 baseline 的二阶段。**

更准确地说：

- 一阶段负责提供内部几何与算子证据
- 二阶段负责把这些结构信号压缩成一个可端到端、可归因、可挂接在已发表宿主上的方法原型

因此当前默认主叙事不再是：

- 继续扩旧的冻结慢层主线
- 继续给 `P0a.1 / R0 / P0b-lite` 加新分支

而是：

**在外部宿主上验证“局部闭式判别残差层”是否成立。**

## 2. 二阶段当前主线

当前二阶段宿主固定为：

- `Tensor-CSPNet`

当前方法原型固定为：

`Tensor-CSPNet backbone -> Temporal_Block 后 latent -> local closed-form residual head -> residual fusion`

实验矩阵固定为三个 arm：

- `E0`
  - 原始 `Tensor-CSPNet`
- `E1`
  - `Tensor-CSPNet + residual linear adapter`
- `E2`
  - `Tensor-CSPNet + local closed-form residual head`

当前协议固定为：

- `BCIC-IV-2a holdout`

当前阶段不做：

- `Graph-CSPNet`
- `CV`
- 第二插入点搜索
- 大量超参矩阵

## 3. 一阶段现在扮演什么角色

一阶段没有失效，但角色已经变了。

当前应把一阶段理解为：

**结构证据层 / 方法学提炼层。**

它当前最重要的输出不是直接当最终模型，而是告诉二阶段三件事：

1. `same/opp bipolar` 比 `same-only` 更对
2. 当前真瓶颈之一是 `global fixed operator scope`
3. 真正值得推进的是 `query-conditioned local operator`

这些信号主要来自：

- [PIA-Operator-当前主线总览.md](/home/THL/project/MTS-PIA/工程记录/PIA-Operator-当前主线总览.md)
- [PIA-Operator-P0a.1-C3-Discriminative-Closed-Form-Probe-promote.md](/home/THL/project/MTS-PIA/工程记录/PIA-Operator-P0a.1-C3-Discriminative-Closed-Form-Probe-promote.md)
- [PIA-Operator-P1a-Zero-Prior-Structured-State-Mapper-Probe-promote.md](/home/THL/project/MTS-PIA/工程记录/PIA-Operator-P1a-Zero-Prior-Structured-State-Mapper-Probe-promote.md)

因此一阶段当前身份应是：

- 内部证据引擎
- 归因来源
- 几何叙事来源

而不是：

- 当前默认对外方法形态

## 4. 当前已经完成到哪一步

### 4.1 外部宿主已复现

`Tensor-CSPNet` 已在本机完整复现到：

- `BCIC-IV-2a holdout = 0.7238`

这意味着它现在可以被视为可信的外部宿主 baseline。

### 4.2 端到端原型已完成最小 smoke

当前已经完成：

- `E0`
- `E1`
- `E2`

在 `subject 1 / 1 epoch / seed 1` 下的最小 smoke。

当前最小结果见 [分类调试记录.md](/home/THL/project/MTS-PIA/工程记录/分类调试记录.md)：

- `E0 = 0.3125`
- `E1 = 0.2604`
- `E2 = 0.4826`

这一步的意义不是宣称方法已成立，而是说明：

- 宿主 wrapper 已通
- 残差融合已通
- 局部闭式残差头的 forward / backward / training loop 已通

### 4.3 当前正在做什么

当前正在执行：

- `S3`
  - `BCIC holdout` 单 seed 全量比较

第一步先跑：

- `E0`

目的不是先看增益，而是先确认：

- 新 wrapper 不会把原始 `Tensor-CSPNet` 宿主改坏
