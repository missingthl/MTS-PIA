# 分类框架全貌

更新时间：2026-03-27

目的：

- 用一份独立文档说明当前分类工程从数据集输入到最终 `trial_acc / macro_f1` 的完整计算链
- 明确当前三条中段路线：
  - `legacy Route B`
  - `unified shell`
  - `PIA Core recovery chain`
- 把真正的方法学核心、外层调度壳、bridge、终端评估器的职责分开

---

## 1. 总体结构

当前工程的共同前端是：

`dataset trials -> protocol split -> SPD/log-Euclidean z-space`

之后分成三条中段路线：

1. `legacy Route B`
2. `unified shell`
3. `PIA Core recovery chain`

再统一进入：

`bridge -> raw MiniROCKET -> window metrics -> trial metrics`

当前最重要的工程判断：

- `raw MiniROCKET` 是唯一终点评判器
- `bridge` 是 raw 域映射器，不是方法学核心
- `unified policy` 是调度壳，不应再被误写成核心创新
- `PIA/TELM2` 的模板学习与显式仿射增强算子，才是原始方法学核心

## 2. 数据集输入与协议切分

统一入口：

- `datasets/trial_dataset_factory.py`

协议切分：

- `scripts/protocol_split_utils.py`

每个 trial 最终会被读成一个字典，核心字段至少包含：

- `trial_id_str`
- `label`
- `x_trial`
- 对 fixed-split 数据集，还会有 `split=train/test`

原始样本记为：

`X_raw ∈ R^(C×T)`

其中：

- `C` = 通道数
- `T` = 当前 trial 的时间长度

当前协议分三类：

1. fixed-split 数据集
   - `HAR`
   - `NATOPS`
   - `FingerMovements`
   - `SelfRegulationSCP1`
   - `BasicMotions`
   - `HandMovementDirection`
   - `UWaveGestureLibrary`
   - `Epilepsy`
   - `AtrialFibrillation`
   - `PenDigits`
   - 直接使用数据集自带 `train/test`

2. `MITBIH`
   - 使用 `npz train/test`

3. `SEED family`
   - `seed1 / seediv / seedv`
   - 使用 native protocol

然后在 official train 内部，再做一次 inner split：

- `train_full -> inner_train + inner_val`

当前硬约束：

- `test` 不参与模板学习
- `test` 不参与 best-round 选择
- `test` 不参与 feedback 更新

## 3. 表示层：raw trial -> SPD/log-Euclidean z-space

核心实现：

- `route_b_unified/representation.py`
- `scripts/run_raw_bridge_probe.py`
- `scripts/run_phase14r_step6b1_rev2.py`

### 3.1 trial 协方差

对每个 raw trial：

`X_raw ∈ R^(C×T)`

先做按时间维中心化：

`X_c = X_raw - mean_t(X_raw)`

然后计算协方差：

`Sigma = (X_c X_c^T) / (T - 1) + eps * I`

其中：

- `Sigma ∈ R^(C×C)`
- `eps` 是 SPD 正则

### 3.2 SPD 对数映射

做特征分解：

`Sigma = U diag(lambda) U^T`

再做对数映射：

`log(Sigma) = U diag(log(lambda)) U^T`

这是 log-Euclidean 表示。

### 3.3 训练集中心化

对 inner-train 全部 trial 的 `log(Sigma)` 求均值：

`L_bar = mean_i log(Sigma_i)`

然后每个 trial 做中心化：

`L_centered = log(Sigma) - L_bar`

### 3.4 上三角向量化

取上三角含对角元素：

`z = vec_utri(L_centered)`

于是得到：

- `X_train ∈ R^(N_train×D)`
- `X_val ∈ R^(N_val×D)`
- `X_test ∈ R^(N_test×D)`

其中：

`D = C(C+1)/2`

这就是当前所有中段模块吃的 `z-space`。

## 4. 当前三条中段路线

### 4.1 legacy Route B

关键词：

- `TELM2 direction bank`
- `additive shift`
- `multiround budget`

核心实现：

- `scripts/run_phase15_step1b_multidir_matrix.py`
- `scripts/run_phase15_multiround_curriculum_probe.py`

#### 4.1.1 direction bank

先在 `X_train ∈ R^(N×D)` 上运行 `TELM2Transformer`，得到：

- `W ∈ R^(K×D)`
- `b ∈ R^K`

然后把 `W` 做：

- 行去均值
- 行归一化

得到：

`direction_bank ∈ R^(K×D)`

#### 4.1.2 single-round 增强

对每个样本，从方向库选一个子集，再随机混合：

`w_mix = sum_j a_j w_j`

然后增强写成：

`z_aug = z + gamma * w_mix`

#### 4.1.3 multiround 增强

curriculum 版本更直接：

- 有每个方向概率 `direction_probs ∈ R^K`
- 有每个方向步长 `gamma_by_dir ∈ R^K`

然后对每个样本抽一个方向，并随机选 `sign ∈ {-1, +1}`：

`z_aug = z + sign * gamma_dir * w_dir`

当前重要判断：

- 这已经不是原始 PIA 的仿射算子
- 这是 direction bank 上的加法扰动机制

---

### 4.2 unified shell

关键词：

- `direction ranking`
- `top-K`
- `weights`
- `gamma budget`
- `feedback update`

核心实现：

- `route_b_unified/policy.py`
- `scripts/run_route_b_unified_probe.py`

当前 `PolicyState` 里的核心矩阵/向量：

- `direction_bank ∈ R^(K×D)`
- `base_scores ∈ R^K`
- `current_scores ∈ R^K`
- `gamma_by_dir ∈ R^K`
- `prior_frozen_mask ∈ {0,1}^K`

#### 4.2.1 初始化

有两类方向源：

1. `legacy_multiround`
   - 直接复用上面的 `TELM2 direction bank`

2. `unified_static / unified_feedback`
   - 使用 `LRAES`

#### 4.2.2 LRAES 的矩阵

`LRAES` 的核心来自每个类别的局部判别矩阵：

- `S_expand ∈ R^(D×D)`
- `S_risk ∈ R^(D×D)`

正则后：

`S_expand_reg = S_expand + lambda I`

`S_risk_reg = S_risk + lambda I`

构造：

`M = S_expand_reg - beta * S_risk_reg`

然后特征分解：

`M = Q diag(eigvals) Q^T`

取主特征向量作为候选方向，拼成新的 `direction_bank`。

#### 4.2.3 policy step

`policy_step` 的核心动作：

1. 用 `current_scores` 在 active directions 上排序
2. 选 top-K
3. 用 softmax-like 归一化成方向权重
4. 结合 `gamma_by_dir` 输出本轮 action

action 至少包含：

- `selected_dir_ids`
- `direction_weights`
- `step_sizes`
- `gamma_vector`
- `entropy`
- `stop_flag / stop_reason`

#### 4.2.4 feedback update

`update_policy` 用 posterior 做轻反馈更新：

`score_(t+1)(d) = score_t(d) + eta_reward * reward_t(d) - eta_penalty * penalty_t(d)`

reward 主要来自：

- 正 margin
- 正 round gain
- 被选中方向权重

penalty 主要来自：

- 负 margin
- flip
- intrusion
- worst-dir
- classwise distortion
- inter-class margin shrink

当前工程判断：

- unified 是调度壳
- 不是原始 PIA 的方法学本体

---

### 4.3 PIA Core recovery chain

关键词：

- `TELM2 closed-form template learner`
- `explicit affine operator`
- `sample-conditional stretch`

核心实现：

- `route_b_unified/pia_core.py`
- `scripts/run_route_b_pia_core_minimal_chain.py`

这是当前为了“把灵魂接回来”新增的主链。

#### 4.3.1 模板学习

输入：

`X_tr ∈ R^(N×D)`

配置：

- 模板数 `L`
- 迭代次数 `n_iters`
- 激活函数
- 岭参数 `C_repr`

先做前向响应：

`P = act(X_tr W^T + b)`

其中：

- `P ∈ R^(N×L)`
- `W ∈ R^(L×D)`
- `b ∈ R^L`

#### 4.3.2 反激活目标

先把 `X_tr` 每一列缩放到激活域，再做反激活：

`Y4 = inv_act(scale(X_tr))`

得到：

`Y4 ∈ R^(N×D)`

#### 4.3.3 闭式回写

求解岭回归：

`A = P^T P + (1/C_repr) I ∈ R^(L×L)`

`B = P^T Y4 ∈ R^(L×D)`

`YYM = A^(-1) B ∈ R^(L×D)`

然后回写：

`W <- YYM`

可选再做：

- 偏置更新
- 行正交化

这是当前 `TELM2/PIA` 的真正闭式核心。

#### 4.3.4 显式仿射增强算子

当前恢复后的增强算子不再是：

`z + gamma * d`

而是：

1. 用训练均值做中心化：

`z_c = z - mu`

2. 用正交化后的模板方向 `d_k` 做投影：

`t_k = <z_c, d_k>`

`comp_k = t_k d_k`

3. 残差：

`residual = z_c - sum_k comp_k`

4. 仿射拉伸：

`z' = mu + sum_k (1 + gamma_k) * comp_k + residual`

当前默认最小版只启用单主模板：

- `gamma = [0.1, 0, 0, ...]`

所以这是：

- sample-conditional
- template-axis affine stretch
- residual preserving

这才是原始 PIA 最接近的语义。

## 5. mechanism / direction health 诊断是怎么算的

核心实现：

- `scripts/run_phase15_step1b_multidir_matrix.py`
- `scripts/run_phase15_multiround_curriculum_probe.py`

这些诊断主要服务于 legacy / unified 外壳。

输入：

- 真实表示 `X_r ∈ R^(N_r×D)`
- 生成增强 `X_g`
- 接受增强 `X_a`
- 增强源样本 `X_s`
- 每个增强对应方向 `dir`

### 5.1 reference classifier

先标准化：

`X_r_scaled = StandardScaler(X_r)`

然后训练参考分类器：

- `LinearSVC` on `X_r_scaled`

### 5.2 true-class margin

对增强前后的样本分别算 decision scores。

定义：

`margin(x, y) = score_y - max_(c != y) score_c`

然后：

- `flip` = margin 符号是否变化
- `margin_drop_median = median(margin_aug - margin_src)`

### 5.3 KNN intrusion

在 `X_r` 上建 `NearestNeighbors`，对每个增强样本查近邻。

定义：

`intrusion = 1 - same_class_neighbor_ratio`

### 5.4 budget 更新

curriculum 的外层预算更新器把：

- `margin`
- `flip`
- `intrusion`

统一归一成安全分数 `safety`，然后把每个方向分到：

- `expand`
- `hold`
- `shrink`
- `freeze`

## 6. bridge：z-space / target -> raw

核心实现：

- `route_b_unified/bridge.py`
- `transforms/whiten_color_bridge.py`

bridge 的输入不是 raw label，而是：

1. 原 trial raw：
   - `x_orig ∈ R^(C×T)`
2. 原协方差：
   - `Sigma_orig ∈ R^(C×C)`
3. 目标增强表示：
   - `z_aug ∈ R^D`

### 6.1 从 z 向量还原目标 SPD

先把 `z_aug` 还原成中心化 log 协方差，再加回训练均值：

`L_aug = unvec(z_aug) + mean_log_train`

然后做矩阵指数：

`Sigma_aug = exp(L_aug)`

### 6.2 whitening-coloring 映射

定义：

`A = Sigma_aug^(1/2) * Sigma_orig^(-1/2)`

对 raw signal 做：

`x_centered = x_orig - mean_t(x_orig)`

`x_aug = A * x_centered + mean_t(x_orig)`

这就是当前 `bridge_single` 的 closed-form 映射。

### 6.3 bridge 输出的指标

重新从 `x_aug` 算经验协方差 `Sigma_emp`，然后计算：

- `bridge_cov_match_error`
- `bridge_cov_match_error_logeuc`
- `bridge_cov_to_orig_distance_logeuc`
- `energy_ratio`
- `cond_A`
- `raw_mean_shift_abs`

再做 classwise 统计：

- `classwise_mean_shift`
- `classwise_covariance_distortion`
- `inter_class_margin_proxy`

其中 `inter_class_margin_proxy` 当前不是分类器 margin，而是类均值协方差间 Frobenius 距离的 proxy。

当前工程判断：

- bridge 是外部 raw 域映射器
- 不是 PIA Core 本体

## 7. raw MiniROCKET 终端评估

核心实现：

- `scripts/run_bridge_curriculum_pilot.py`
- `scripts/run_raw_minirocket_baseline.py`

输入：

- `train_trials`
- `test_trials`

### 7.1 窗口策略

先决定窗口策略：

1. 长序列数据集
   - 用固定秒数窗口

2. 短序列数据集
   - 用 `train` 中位长度 `T_med` 的比例确定窗口长度和 hop

输出：

- `window_len_samples`
- `hop_len_samples`

### 7.2 建窗口

对每个 raw trial 拆成窗口：

`X_win ∈ R^(N_windows×C×T_w)`

每个窗口做：

- per-window
- per-channel
- z-score normalization

训练集窗口会做 cap，防止某些长 trial 产生过多窗口。

### 7.3 MiniROCKET

模型：

- `aeon MiniRocketClassifier`

先在窗口层预测，再聚合回 trial：

- `majority`
或
- `meanlogit`

最终输出：

- `window_acc`
- `window_macro_f1`
- `trial_acc`
- `trial_macro_f1`

当前统一结论：

- `raw MiniROCKET` 是唯一终点评判器
- 但它不应反过来定义 PIA 的数学本体

## 8. 当前框架里最重要的矩阵清单

从 raw 到 final，当前会出现这些关键矩阵：

1. 原始 trial：
   - `X_raw ∈ R^(C×T)`

2. 协方差：
   - `Sigma ∈ R^(C×C)`

3. log 协方差：
   - `log(Sigma) ∈ R^(C×C)`

4. z-space 向量：
   - `z ∈ R^D`

5. 训练集表示矩阵：
   - `X_train ∈ R^(N×D)`

6. TELM2 模板矩阵：
   - `W ∈ R^(L×D)`

7. TELM2 响应矩阵：
   - `P ∈ R^(N×L)`

8. 反激活目标：
   - `Y4 ∈ R^(N×D)`

9. 岭回归正规方程：
   - `A = P^T P + (1/C)I ∈ R^(L×L)`
   - `B = P^T Y4 ∈ R^(L×D)`
   - `YYM = A^(-1)B ∈ R^(L×D)`

10. legacy / unified 方向库：
   - `direction_bank ∈ R^(K×D)`

11. LRAES 判别矩阵：
   - `S_expand ∈ R^(D×D)`
   - `S_risk ∈ R^(D×D)`
   - `M = S_expand_reg - beta * S_risk_reg ∈ R^(D×D)`

12. bridge 映射矩阵：
   - `A = Sigma_aug^(1/2) Sigma_orig^(-1/2) ∈ R^(C×C)`

13. raw window 张量：
   - `X_win ∈ R^(N_w×C×T_w)`

## 9. 当前工程状态的一句话判断

当前工程不是一条单纯的线性主链，而是：

- 一个共同的 `raw -> SPD/log -> z-space` 表示前端
- 三种不同的中段哲学：
  - `legacy additive shift`
  - `unified policy shell`
  - `PIA Core affine recovery`
- 一个共同的 `bridge -> raw MiniROCKET` 外部验证后端

如果从方法学核心看：

- 当前真正应该被视为核心的是：
  - `TELM2 template learning`
  - `PIA affine operator`

如果从工程壳看：

- 当前还保留着：
  - `policy`
  - `bridge`
  - `evaluator`

因此现在的工程本质上处于：

- `unified shell` 仍存在
- 但 `PIA Core` 已经被重新拉回来的过渡阶段
